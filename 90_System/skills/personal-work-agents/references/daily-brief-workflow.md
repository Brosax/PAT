# Daily Brief Workflow

Use this reference for morning introductions and end-of-day summaries.

## Shared Rules

- Read `40_Tasks/todo.md` first.
- Interpret today in `Europe/Madrid`.
- Use absolute dates for due dates and overdue items.
- Output to the current Codex thread by default.
- Do not create daily archive files unless the user explicitly asks.

## Morning Brief

Default schedule: weekdays at 08:00 Europe/Madrid.

Include:

- A short greeting and date.
- Today's focus based on due dates, `Doing`, and high-signal `Todo` items.
- Due today and overdue tasks.
- Waiting items that may need a nudge.
- Three recommended first actions.
- Questions for missing deadlines or ambiguous priorities, only if they materially affect the day.

Keep it concise and practical. Prefer a handful of bullets over a long report.

## End-Of-Day Summary

Default schedule: weekdays at 16:30 Europe/Madrid.

Include:

- What appears completed from `Done` items updated today.
- Still-open `Doing`, `Todo`, and urgent `Inbox` items.
- Waiting items and blockers.
- Suggested tomorrow priorities.
- Task or memory candidates that need user confirmation.

If there is not enough information to know what was completed, say so and invite the user to mark completed items by id.

## Tone

Use calm, direct Chinese. Keep the brief useful enough to act on in under one minute.
