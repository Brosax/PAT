## Required Pre-Update Gate

Before writing to the PPT:

1. Read the latest or previous OST slide content from the current deck.
2. Summarize the existing items back to the user.
3. Ask how each item is progressing, whether there are blockers, and whether the status should move.
4. Wait for the user's status/progress input or explicit confirmation.

If the user has not provided current progress/status text, do not update the PPT.

## Update Rule

When the user provides status/progress:

- Preserve the existing goals and wording where possible.
- Add the user's status/progress as an update layer.
- Do not rewrite goals from scratch.
- Keep the deck language in English unless the user explicitly asks otherwise.
- If a status changes, record the new status and preserve the prior context when useful.

## Suggested Status Questions

Ask concise questions such as:

- Which items progressed since the last OST slide?
- Should any item move from `Planned` to `Ongoing`, `Waiting`, `Blocked`, or `Done`?
- What short status text should be shown for each changed item?
- Are there blockers or next steps that should appear in the slide?
